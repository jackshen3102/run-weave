Playwright 负责页面行为，独立 raw CDP connection 负责 Runweave custom domain。两条通道共享 scoped Browser group 和物理 debugger，但不要求同一 connection 为同 target 创建第二个逻辑 session。

冻结决策限制是设计，不是缺陷。

Agent 通过 root Target.attachToTarget 取得 primary session；不使用 browserContext.newCDPSession(page)。

跨边界调用

root/browser custom command、跨 group target、未知 session、85% 与额外 targetId 都不得修改状态。

为什么 command 不带 targetId

target 在 attach 时已由 scoped connection 校验；后续 command 只相信 session ownership。

为什么不用 newCDPSession

它要求同 target 的第二个 logical session，超出当前 proxy 的身份与事件模型。

为什么 raw connection 合理

它走现有 root attach 合约，不修改 proxy cardinality；Playwright 仍独立负责页面语义。
