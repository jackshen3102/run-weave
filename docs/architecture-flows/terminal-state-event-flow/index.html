Terminal state & event architecture

一个终端，
不是一个状态。

当前代码里至少有六条并行状态轴。进程是否存在、连接是否可用、Agent 是否工作、Thread 是否真实运行、是否需要用户注意，以及多 Pane 如何汇总，必须分开判断。

First principle 事件只负责“发生了什么”，投影才负责“现在是什么”。

绿点和铃铛属于注意力事件；它们不能替代进程、连接或 Agent 状态。恢复时优先回到权威快照，再继续消费增量。

它们可以同时取不同值。例如：进程 running、连接 connecting、Agent running、Thread running、completion 绿点仍亮。
