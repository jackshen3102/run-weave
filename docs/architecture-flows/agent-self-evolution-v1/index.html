03 / PROMPT INTELLIGENCE

Prompt 是关键输入，但不能成为产品补丁

系统保留完整的 Prompt 因果链，重点比较“原始表达 → 理解 → 行为 → 纠偏 → 结果”，再判断真正责任层。

原则 · Prompt 是加速器，不是正确性补丁

Prompt 质量不是“更长”，而是信息结构更稳

同一意图需要用简短、自然、专业、含噪四种表达做鲁棒性测试，避免把偶然有效的“魔法短语”沉淀成经验。

04 / AUTONOMY LEVELS

L0–L4：能力可以到四级，权限必须逐级放开

L4 不是“无约束自治”。它只允许在本地、可逆、经过证据门禁的范围内自动使用和维护已晋级资产。

目标上限 · L4 Local Autonomy
05 / OPERATING MODEL

日更抽取、周更晋级、月度退休

原始内容只有 7 天保留窗口，因此 Episode 不能只按周生成；周报只是输出节奏，不应成为数据处理节奏。

关键约束 · Raw content retention = 7d

不可越过的安全边界

  • 外部发布、自动合并、提权与破坏性操作始终需要人工授权
  • 每个资产保留来源 Episode、评估记录、版本与回滚点
  • 结论按项目、任务、模型与版本设适用范围，不做无证据泛化
  • 用户删除原始记录后，派生资产必须降置信度或重新计算
  • 数据缺口、保留期截断和小样本必须显式影响结论置信度

四个核心数据合约

把“报告文字”升级成可追溯、可评估、可注入的实体。

多维智能，不压成一个总分

每次实验输出指标向量和回归项；只有在目标维度改善且关键维度不退化时才晋级。

性质：技术架构方案，不是 Runweave 产品 UI，也不代表目标能力已经实现。 范围：个人本地 V1；团队共享、云端训练与跨用户经验交换不在本方案内。
