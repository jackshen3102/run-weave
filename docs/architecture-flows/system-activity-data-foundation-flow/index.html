Independent activity data plane

先把真实行为独立沉淀，再让模型从证据中生成经验

Stable、Beta、Dev 与 external producer 在行为发生时主动写入同一个 Activity Hub。 Facts 是系统记录，Learning 是模型派生；两者分开存储、分开审计、分开过期。

ARCHITECTURE BOUNDARY 不是从现有 Thread、日志或 Agent Team 文件临时汇总。

大多数数据写入 Activity Hub 自己的结构；只有完整 Thread、scrollback、run、截图等少量大对象保存引用。
