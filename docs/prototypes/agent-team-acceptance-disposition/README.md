# Agent Team Acceptance Case 人工裁决原型

面向 Agent Team 环境跳过与不合理 Case 的可运行 HTML 交互原型。

## 启动

```bash
python3 -m http.server 6188 --directory docs/prototypes/agent-team-acceptance-disposition
```

打开 `http://127.0.0.1:6188/`。

## 原型简报

- 目标：模型无法通过 Case 时，由人工确认环境问题或 Case 不适用，解除 Run 阻塞。
- 用户动作：填写裁决理由，选择“确认环境问题并跳过”或“标记 Case 不适用”。
- 主要用户：正在处理 Agent Team `need_human` 状态的操作者。
- 影响界面：终端右侧 Agent Team 面板。
- 关键流程：查看不可变观察 → 提交人工裁决 → Case 显示已裁决 → Run 以例外结果完成。
- 重要状态：等待裁决、提交中、已完成且有人工裁决。
- 非目标：不创建 follow-up work item，不把 skipped/fail 改写成 pass，不提供通用强制完成。

## 验证点

- 首屏只显示产品需要的 Case、观察事实、理由输入和裁决操作。
- 没有填写理由时不能提交。
- 裁决后仍显示原始“环境跳过”事实，同时显示人工裁决结果。
- 最终状态是“已完成 · 有人工裁决”，不是纯成功。

## 功能分类

### 产品核心功能

| 元素 / 行为                 | 最终产品是否需要 | 产品价值                 | 备注                           |
| --------------------------- | ---------------- | ------------------------ | ------------------------------ |
| 最新 observation 与环境原因 | 是               | 人工基于事实裁决         | 不可被 disposition 覆盖        |
| 必填裁决理由                | 是               | 保留审计依据             | 空理由禁用提交                 |
| 确认环境问题并跳过          | 是               | 环境阻塞不再否决整个 Run | 仅环境 skip 可用               |
| 标记 Case 不适用            | 是               | 不合理验收合同可人工解除 | 不等价于 pass                  |
| 已完成且有人工裁决          | 是               | 区分纯成功和例外完成     | 对应 completed_with_exceptions |

### 原型辅助功能

无。刷新页面恢复初始 mock 状态，不进入产品界面。

## 冻结记录

- 最终采用的交互：在 `need_human` 面板顶部展示单个待裁决 Case，理由必填后提交结构化 disposition。
- 放弃的方向：通用“人工完成”按钮；把 Case 状态改成 pass；尚无真实 work item 的“转 follow-up”。
- 产品核心功能清单是否已确认：是。
- 原型辅助功能清单是否已确认：是。
- 最终截图：浏览器验收后按需补充。
- 冻结时间：2026-07-19。

## 边界

- 原型不连接真实后端，也不导入生产源码。
- disposition 必须绑定当前 observation；重新执行后旧裁决不能继续生效。
- 原型不能证明协议、存储或运行时已经支持该能力。

## 实施计划衔接

- 原型表达的产品行为：人工可以带理由解除环境 skip 或不合理 Case 的阻塞。
- 需要进入产品实现的核心功能：共享 disposition 合同、后端裁决 API、统一 completion evaluator、面板裁决卡片。
- 不进入产品实现的原型辅助功能：无。
- 需要检查的现有代码：finding disposition、completion policy、Agent Team panel。
- 验收方式：YAML 合同、后端 verifier、类型检查、lint、真实 Agent Team 页面 Playwright 验收。
