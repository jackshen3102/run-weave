# Agent Team 当前关注卡片

面向 Agent Team 执行侧栏的可运行 HTML 交互原型。

## 启动

```bash
python3 -m http.server 6188 --directory docs/prototypes/agent-team-attention-summary
```

打开：

```text
http://127.0.0.1:6188/
```

## 文件

- `index.html`：静态页面壳层、样式和挂载点。
- `app.js`：查看详情与聚焦 pane 的原型交互。
- `mock-state.json`：模拟 Run、Review blocker 和验收 Case。
- `prototype-preview.png`：冻结后的浏览器验证截图。

## 原型简报

- 目标：无论下方用例和日志多长，用户都能在顶部立即看到当前最主要的问题。
- 用户动作：查看问题摘要、定位到底部关联 Case、聚焦对应 Code pane。
- 主要用户：观察或接管 Agent Team Run 的开发者。
- 影响的产品界面或模块：Agent Team 右侧执行面板。
- 关键流程：读取顶部当前关注卡片 → 定位详情或聚焦修复 pane。
- 重要状态：需要人工介入、Review 阻断、Case 未通过、Run 完成但有遗留。
- 非目标：不增加分类导航、问题抽屉、三栏仪表盘或新的后端接口。

## 验证点

- 当前关注卡片位于滚动区外，下方内容滚动后仍保持可见。
- 默认展示最高优先级问题的类型、严重级别、摘要、关联 Case 和处理位置。
- “查看详情”定位到关联失败 Case。
- “聚焦 Code pane”是最终产品保留的入口，不是原型辅助按钮。
- Run 已结束但仍有未通过项时，顶部不显示纯“已完成”。

## 功能分类

### 产品核心功能

| 元素 / 行为          | 最终产品是否需要 | 产品价值                 | 备注                    |
| -------------------- | ---------------- | ------------------------ | ----------------------- |
| 固定顶部当前关注卡片 | 是               | 长内容下仍能看到首要问题 | 只在存在问题时显示      |
| 问题类型、摘要与数量 | 是               | 说明卡点和影响           | Review 与关联 Case 去重 |
| 聚焦修复 pane        | 是               | 快速进入处理位置         | 复用现有 focus pane API |
| 查看关联详情         | 是               | 快速核对证据             | 定位到现有 Case 卡片    |
| 有遗留的结束态       | 是               | 避免“已完成”与失败项冲突 | 不改变后端 Run 状态     |

### 原型辅助功能

| 元素 / 行为                        | 辅助验证什么   | 为什么不进入产品              | 备注                  |
| ---------------------------------- | -------------- | ----------------------------- | --------------------- |
| `?state=` URL 参数                 | 切换 mock 状态 | 产品从真实 Run 状态计算       | 默认不显示任何辅助 UI |
| 点击聚焦后写入 `data-focused-pane` | 证明入口可触发 | 真实产品会调用 focus pane API | 无用户可见提示        |

## 冻结记录

- 最终采用的交互：滚动区外固定当前关注卡片；可定位关联 Case 或聚焦 Code pane。
- 放弃的方向：三类状态卡、问题收件箱、标题栏抽屉、概览导航。
- 产品核心功能清单是否已确认：是。
- 原型辅助功能清单是否已确认：是。
- 最终截图：`prototype-preview.png`；不包含 prototype helper。
- 冻结时间：2026-07-15 00:26 CST。

## 边界

- 这个原型不连接真实后端 API。
- 这个原型不导入生产源码。
- 这个原型不能证明产品协议、存储或运行时支持已经存在。
- 原型通过 URL 参数切换 mock 状态，不向产品画面添加演示控件。

## 实施计划衔接

- 原型表达的产品行为：异常聚合区固定在面板头部，提供摘要和直接处理入口。
- 需要进入产品实现的核心功能：前端聚合、Review/Case 去重、状态标签、定位详情、聚焦 pane。
- 不进入产品实现的原型辅助功能：mock 状态和 `data-focused-pane`。
- 已确认支持的能力：`AgentTeamRun.status`、`acceptance`、`loop.repairCycles`、`focusAgentTeamPane`。
- 需要检查的现有代码：`terminal-agent-team-panel.tsx`、`terminal-agent-team-panel-sections.tsx`、`terminal-agent-team-panel-model.ts`。
- 可能涉及的协议或数据结构：无；现有 Run 字段足够表达顶部摘要。
- 可能涉及的前端落点：面板滚动区外的关注卡片，以及失败 Case 的定位标识。
- 可能涉及的后端或运行时落点：无。
- 验收方式：Playwright 验证卡片固定、详情定位和最终真实页面行为。
